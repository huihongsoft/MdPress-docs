核心概念

Tags: 指南 概念

要精通 MdPress，理解其背后的核心概念至关重要。

1. Markdown 为真理之源 (Source of Truth)

在 MdPress 中，Markdown 文件不仅仅是内容，它们就是网站本身。

  • 文件名即 URL: docs/guide/intro.md 自动对应 yoursite.com/guide/intro.html。
  • 文件夹即分类: 文件夹结构直接决定了 URL 路径结构。

2. 静态站点生成 (SSG)

MdPress 不依赖数据库。它在构建时 (Build Time) 遍历所有 Markdown 文件，根据主题模板预先渲染出纯 HTML/CSS/JS 文件。 这意味着：

  • 高性能: 访问网站时无需服务器端计算，直接加载静态资源。
  • 高安全: 没有数据库注入风险。
  • 低成本: 可以部署在任何廉价甚至免费的静态托管服务上（GitHub Pages, Vercel 等）。

3. Frontmatter 元数据

每个 Markdown 文件的顶部都可以包含一段 YAML 格式的配置块，称为 Frontmatter。 它是内容与配置的桥梁。您可以在这里定义：

  • 页面标题和描述（用于 SEO）
  • 布局模式（主页、文档、自定义）
  • 侧边栏显隐
  • 特殊组件配置（如 Steps 步骤条）

4. 主题驱动 (Theme Driven)

MdPress 的界面完全由主题决定。

  • 内容层: Markdown 文件。
  • 表现层: 主题（HTML 模板 + CSS）。 更换主题就像给手机换壳一样简单，不会影响您的任何内容数据。

5. 插件系统

MdPress 采用插件化架构。

  • 核心: 仅负责文件解析和基础构建。
  • 扩展: PDF 导出、Word 导出、数学公式渲染等功能均通过插件实现。 这种设计保证了核心的轻量与稳定，同时提供了无限的扩展可能。
