# 长文写作最佳实践

MdPress 专为处理大型文档和长篇文章而设计。以下是一些帮助您更好组织内容的建议。

## 1. 合理使用标题层级

Markdown 的 `#` 标题不仅决定了字体大小，还决定了文章的结构和大纲。

- **H1 (`#`)**: 文章标题，每页仅应有一个。
- **H2 (`##`)**: 主要章节。
- **H3 (`###`)**: 子章节。

MdPress 的 Official 主题会自动根据 H2 和 H3 生成右侧的 **本页大纲 (On this page)**，方便读者快速跳转。

## 2. 开启右侧大纲

对于长文，右侧大纲是导航的关键。确保在 Frontmatter 中开启了它（默认开启）：

```yaml
---
outline: deep # 显示 H2 和 H3
---
```

或者在全局 `config.json` 中配置：

```json
{
  "themeConfig": {
    "outline": [2, 3]
  }
}
```

## 3. 使用自定义容器强调重点

长文中容易产生视觉疲劳，使用自定义容器可以打破单调的布局，突出重要信息。

```markdown
::: tip 提示
这是一个关键知识点。
:::

::: warning 注意
请避免这种常见的错误配置。
:::
```

## 4. 代码块折叠

如果您的文章包含大量代码，建议使用代码折叠功能（如果主题支持）或将长代码拆分为多个片段，配合说明文字。

## 5. 拆分长文档

如果一篇文章超过 5000 字，或者包含多个独立的主题，建议将其拆分为多个 `.md` 文件，并使用侧边栏进行组织。

- **Bad**: `guide.md` (包含安装、配置、使用、部署所有内容)
- **Good**:
  - `guide/installation.md`
  - `guide/configuration.md`
  - `guide/usage.md`
  - `guide/deployment.md`

这样不仅利于维护，也能提升读者的阅读体验。
