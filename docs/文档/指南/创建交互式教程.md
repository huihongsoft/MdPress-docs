# 创建交互式教程

MdPress 引入了一种创新的文档组件 —— **Steps (步骤条)**。它允许您将复杂的安装指南、操作流程拆分为交互式的步骤，极大地提升了用户的阅读体验。

## 什么是 Steps？

Steps 组件通常用于以下场景：
- **多平台安装指南**: Windows / macOS / Linux 用户只需查看与自己相关的命令。
- **多语言代码示例**: 无论是 Python、JavaScript 还是 Go，用户可以切换查看。
- **分步操作流程**: 将冗长的教程拆解为 Step 1, Step 2...

## 如何使用？

Steps 的数据定义在 Markdown 文件的 Frontmatter 中，通过 `steps` 字段配置。

### 基础结构

```yaml
---
steps:
  - id: step1
    kind: tab     # 类型：tab (选项卡) 或 list (列表)
    title: 第一步标题
    tabs: [...]   # 如果 kind 是 tab
  - id: step2
    kind: list
    title: 第二步标题
    items: [...]  # 如果 kind 是 list
---
```

### 示例 1：多平台安装 (Tab 类型)

这是最常见的用法。用户点击不同的 Tab，下方的内容会随之切换。

```yaml
steps:
  - id: install
    kind: tab
    title: 1. 安装客户端
    tabs:
      - id: win
        label: Windows
        description: |
          下载 `.exe` 安装包并双击运行。
          ```powershell
          winget install mdpress
          ```
      - id: mac
        label: macOS
        description: |
          下载 `.dmg` 文件并将应用拖入 Applications。
          ```bash
          brew install --cask mdpress
          ```
```

### 示例 2：操作清单 (List 类型)

适用于线性的任务列表，每一项都有标题和详情。

```yaml
steps:
  - id: config
    kind: list
    title: 2. 环境配置
    items:
      - title: 设置环境变量
        details: |
          请将 `BIN_PATH` 添加到您的系统变量中。
      - title: 验证安装
        details: |
          运行 `mdpress --version` 检查版本。
```

## 在正文中引用

定义好 Frontmatter 后，MdPress 会自动在页面顶部渲染 Steps 组件。正文部分（Markdown 内容）可以继续编写通用的说明、注意事项或总结。

::: tip 最佳实践
将差异化的操作步骤放在 Steps 组件中，将通用的原理说明放在正文中。
:::
